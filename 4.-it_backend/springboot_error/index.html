SpringBoot 2.7.6 #

__ 01_JSP error #

  • SpringBoot는 JSP를 권장하지 않는다라는 구글에서 보았지만 살포시 무시했다. 그결과 JSP error가 떴다. 이 문제로 2일을 소비했다.

  • 다음부터는 SpringBoot에서 JSP를 절대 안 쓰는 걸로 하고, 이를 해결하기 위해 검색을 했다. (GPT4에게 아무리 물어봐도 똑같은 답변만 나오고 딱히 이상이 없다고 한다.) 처음에는 위와 같은 경로에서 JSP가 정상적으로 작동했다. 하지만 dependency injection을 하면서 갑자기 jsp가 작동하지 않았다.

  • 1차 해결과정 - dependency 추가

    tomcat-embed-jasper는 JSP 파일을 servlet으로 변환하는 컴파일러이다. jasper의 역할은 JSP를 컴파일하는것인데, 내장된 tomcat은 그 역할을 할 수 없기에 추가해 줘야한다고 해서 추가했다.

    해결이 되지 않았다.

<dependency> <groupId>org.apache.tomcat.embed</groupId> <artifactId>tomcat-embed-jasper</artifactId> </dependency>
  • 2차 해결과정 - 구글 다시 검색
**29.4.5 JSP Limitations**  
When running a Spring Boot application that uses an embedded servlet container (and is packaged as an executable archive), there are some limitations in the JSP support.  
**- With Jetty and Tomcat, it should work if you use war packaging.** **An executable war will work** when launched with java -jar, and will also be deployable to any standard container. **JSPs are not supported when using an executable jar.**  
- Undertow does not support JSPs.  
- Creating a custom error.jsp page does not override the default view for error handling. Custom error pages should be used instead.  
There is a JSP sample so that you can see how to set things up.
  • 위와 같은 내용을 확인하고 WAR를 추가
plugins {  
    id 'org.springframework.boot' version '2.3.4.RELEASE'  
    id 'io.spring.dependency-management' version '1.0.10.RELEASE'  
    id "com.ewerk.gradle.plugins.querydsl" version "1.0.10"  
    id 'java'  
    id 'war'
}  
   _OR_  
group = 'com.example'  
version = '0.0.1-SNAPSHOT'  
apply plugin: 'war'

하지만 그래도 안 된다. ㅠㅠㅠ

이것 때문에 2일이 또 소비되었다. 코드를 계속 보는 것도 복습이라고 생각했지만, 정말 답답한 상황….

  • 1차 해결과정

    • 분명히 웹 화면에서 form으로 데이터를 전송했는데, db에는 null값이 저장되었다.
    • 그래서 계속 테스트를 진행했지만 여전히 안 되었다.
    • GPT에서는 Entity가 제대로 기입되지 않아서 생긴 문제라고 해서 혹시나 하고 다시 작성했지만 여전히 안 되었다.

  • 2차 해결과정

    • 우선 데이터가 null 값이지만 적재 되는 것으로 보아 Entity 문제는 아니었다.
    • 아무리 생각해봐도 무엇이 문제인지 몰라서 어디서 null값이 들어가는지 확인하고 싶었다.
    • 그래서 구글 검색으로 log를 통해서 sql 값을 확인했다.
spring.mvc.throw-exception-if-no-handler-found=true
logging.level.org.springframework.web=TRACE
logging.level.com.bit.allsimple.controller=TRACE
logging.level.org.hibernate.SQL=TRACE
logging.level.org.hibernate.type.descriptor.sql=TRACE

  • log를 보니 역시 view에서 db로 넘어가는 순간 null값이 들어가 있다.
  • 그래서 view가 문제라고 생각해서 개발자 도구를 켜고 data가 정상적으로 넘어가는지 확인했다.
  • 역시 view에서는 정상적으로 데이터가 넘어가고 있었다.
  • 3차 해결과정
    • 분명 jsp가 문제였다. jsp가 정상 작동하지 않은 것 같은데, 아무리 gpt에 물어봐도 문제가 없다고 한다…..gpt….
    • 계속 시간을 소비하는 가운데, 정상 작동하는 jsp 예제와 내가 작성한 jsp를 비교했다.
