Linux Kvm

KVM이란? (Ubuntu 20.04) #

__ 1.1_KVM 패키지 설명 #

  • qemu-kvm
    • KVM 하이퍼바이저에 대한 하드웨어 에뮬레이션을 제공
    • Kernel Space - KVM
    • User Space - QEMU
  • libvirt-daemon-system
    • libvirt 데몬을 시스템 서비스로 실행하는 구성 파일
    • 리눅스에서 Hypervisor 기반 가상화 관리 및 제어도구
  • libvirt-clients
    • 가상화 플랫폼을 관리하기위한 소프트웨어
  • bridge-utils
    • 이더넷 브리지를 구성하기위한 명령 줄 도구 세트
  • virtinst
    • 가상 머신을 만들기위한 명령 줄 도구 집합
  • virt-manager
    • 사용하기 쉬운 GUI 인터페이스와 libvirt를 통해 가상 머신을 관리하기위한 명령 줄 유틸리티

__ 1.2_KVM 설치 #

sudo apt-get update sudo apt-get install qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils

__ 1.3_Package Status #

  • Qemu-kvm 상태 확인 명령어
    • sudo systemctl status qemu-kvm
      • sudo systemctl start qemu-kvm (시작)
      • sudo systemctl stop qemu-kvm (정지)

  • KVM-libvirtd 상태 확인 명령어
    • sudo systemctl status libvirtd

__ 1.4_Libeirtd Network #

__ __ 1.4.1_Network Type #

  • NAT
    • 가상머신 개별 IP를 가지지만 외부로 나갈 때는 NAT으로 인터넷이 가능하다.
  • Bridge
    • 호스트의 IP를 공유하는 방식
  • Host-Only
    • 호스트 VM과 게스트 VM은 통신이 가능하지만, 게스트 VM 끼리는 통신이 불가능하다.
  • Internal Networking
    • 가상 머신들이 호스트 시스템과는 독립적인 개인 네트워크에 연결되도록 한다. 가상 머신끼리 통신은 연결이 가능하지만 외부 통신은 불가능하다.

__ __ 1.4.2_Bridge Network #

  • /etc/netpaln/01-network-manager-all.yaml 수정

  • /etc/netplan/br0.xml 작성

  • sudo netplan apply 실행

  • sudo ifconfig 확인

  • sudo virsh net-define /ect/netplan/br0.xml

  • sudo virsh net-list –all 명령어로 br0 네트워크 확인

  • sudo virsh net-start br0 명령어로 br0 네트워크 활성화

  • sudo virsh net-autostart br0 명령어로 br0 네트워크 오토스타트 활성화

  • 명령어 IP addr로 네트워크 확인

    • w로 시작하는 네트워크는 무선 네트워크를 의미(Bridge를 생성이 불가능하다.)
    • e로 시작하는 네트워크는 유선 네트워크를 의미(Bridge 생성이 가능하다.) 
# /etc/netpaln/01-network-manager-all.yaml  수정

network:
  version: 2
  renderer: networkd
  ethernets:
    enx00e04c362038:
      dhcp4: no
  bridges:
    br0:
      interfaces: [enx00e04c362038]
      addresses: [192.168.77.29/24]
      gateway4: 192.168.77.1
      nameservers:
        addresses: [168.126.63.1, 168.126.63.2]
      parameters:
        stp: false
        forward-delay: 0

# /etc/netplan/br0.xml 작성
# sudo vi /etc/br0.xml
<network>
  <name>br0</name>
  <forward mode='bridge' />
  <bridge mode='br0' />
</network>
  • Virsh 명령어
    • network
      • virsh net-list
        • 가상 네트워크 조회
      • virsh net-define Name
        • 가상 네트워크 등록
      • virsh net-undine Name
        • 가상 네트워크 삭제
      • virsh net-destroy Name
        • 가상 네트워크 종료
      • virsh net-start Name
        • 가상 네트워크 시작
      • virsh net-autostart Name
        • 가상 네트워크 자동 시작
  • service libvirtd restart
    • 가상 네트워크 설정 후 재시작

__ 1.5_KVM Create #

  • virt-install 설치

virt-install \
--name test \
--ram 3072 \
--vcpus 2 \
--disk path=/var/lib/libvirt/images/test.qcow2,size=30 \
--os-variant centos7.0 \
--os-type linux \
--network bridge=br0 \
--graphics none \
--console pty,target_type=serial \
--location /img/CentOS-7-x86_64-Minimal-2009.iso  \
--extra-args 'console=ttyS0,115200n8 serial'
  • 스냅샷 만들기
qemu-img craete -f qcow2 -b /var/lib/libvirt/images/test.qcow2  clone.qcow2

virt-install --name k8s_node1 \
--vcpus 2 \
--ram 3082 \         
--disk path=/img/k8s_node1.qcow2 \    
--network bridge=br0,mac=00:00:00:00:00:61 \         
--graphics none \          
--import
  • 용량 조절하기
  1. KVM 내부 저장공간 축소
resize2fs /dev/sda1 10G
  1. qcow2 이미지 축소
virt-sparsify --compress original.qcow2 smaller.qcow2
  1. KVM 내부 저장공간 늘리기
virsh start <가상머신이름>

qemu-img resize <디스크파일경로> +20G
  1. 가상 머신 내부 (참고 )
pvresize /dev/sdaX

lvextend -l +100%FREE /dev/mapper/<볼륨그룹명>-<논리볼륨명>

resize2fs /dev/mapper/<볼륨그룹명>-<논리볼륨명>

__ 1.7_KVM connect #

  • KVM 리스트 확인
    • virsh list –all
  • KVM 콘솔 접속
    • virsh console work_node_1
  • KVM 삭제
    • virsh undefine work_node_1
  • KVM 정지
    • virsh destroy work_node_1
  • KVM 시작
    • virsh start work_node_1
  • KVM 스냅샷

__ 1.8_console 접속 안 될 때 #

  • systemctl enable serial-getty@ttyS0.service
  • systemctl start serial-getty@ttyS0.service

__ 1.9_KVM 복제 #

1. 기존 가상 머신 이미지 복사 #

먼저, 기존 가상 머신(VM)의 디스크 이미지를 복사해야 합니다. 이를 위해 가상 머신이 저장된 위치로 이동합니다. 기본적으로 KVM의 이미지는 /var/lib/libvirt/images/에 저장됩니다.

bashCopy code

cd /var/lib/libvirt/images/

다음으로, cp 명령어를 사용하여 기존 가상 머신의 디스크 이미지를 새로운 이름으로 복사합니다. 예를 들어, original_vm.img를 cloned_vm.img로 복사하는 경우:

bashCopy code

sudo cp original_vm.img cloned_vm.img

2. 복제할 가상 머신 정의 생성 #

기존 가상 머신의 정의를 복사하여 새 가상 머신에 사용할 수 있도록 합니다. 먼저, 기존 가상 머신의 정의를 XML 파일로 내보냅니다.

bashCopy code

virsh dumpxml original_vm > original_vm.xml

XML 파일을 편집하여 새 가상 머신에 맞게 수정합니다. 주요 변경 사항은 가상 머신의 이름과 디스크 이미지 파일 경로입니다. 예를 들어, nano 편집기를 사용할 수 있습니다.

bashCopy code

sudo nano original_vm.xml

  • <name> 태그의 값을 새 가상 머신의 이름으로 변경합니다.
  • <source file=...> 태그(디스크 섹션 내부에 위치)의 file 속성 값을 새로운 이미지 파일 경로로 변경합니다.
  • 필요한 경우 MAC 주소(<mac address=...>)를 제거하거나 변경하여 네트워크 충돌을 방지합니다.

변경 후, 파일을 저장하고 편집기를 종료합니다.

3. 새 가상 머신 정의 등록 #

수정된 XML 파일을 사용하여 새 가상 머신을 정의합니다.

bashCopy code

virsh define original_vm.xml

4. 새 가상 머신 시작 #

새로 정의된 가상 머신을 시작합니다.

bashCopy code

virsh start new_vm_name

여기서 new_vm_name은 XML 파일에서 설정한 가상 머신의 이름입니다.

주의 사항 #

  • 복제 과정에서 네트워크 설정(특히, MAC 주소)과 호스트 이름을 적절히 변경하는 것이 중요합니다. 이는 네트워크 충돌을 방지하기 위함입니다.
  • 가상 머신의 복제는 저장소와 리소스 사용량 증가로 이어질 수 있으므로, 시스템 용량을 사전에 확인하는 것이 좋습니다.
  • 복제된 가상 머신에서 특정 애플리케이션이나 서비스를 실행하기 전에, 해당 서비스의 구성이 새 환경에 적합한지 검토해야 합니다.

이 과정을 통해 Ubuntu 20.04 시스템에서 KVM 가상 머신을 성공적으로 복제할 수 있습니다.
