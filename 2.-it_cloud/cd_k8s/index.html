__ __ Common #

  1. Set SELinux to permissive mode:

    These instructions are for Kubernetes 1.29.

    # Set SELinux in permissive mode (effectively disabling it)
sudo setenforce 0
sudo sed -i 's/^SELINUX=enforcing$/SELINUX=permissive/' /etc/selinux/config

  2. Add the Kubernetes yum repository

# This overwrites any existing configuration in /etc/yum.repos.d/kubernetes.repo
cat <<EOF | sudo tee /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://pkgs.k8s.io/core:/stable:/v1.29/rpm/
enabled=1
gpgcheck=1
gpgkey=https://pkgs.k8s.io/core:/stable:/v1.29/rpm/repodata/repomd.xml.key
exclude=kubelet kubeadm kubectl cri-tools kubernetes-cni
EOF

  1. Install kubelet, kubeadm and kubectl:

    sudo yum install -y kubelet kubeadm kubectl --disableexcludes=kubernetes

  2. (Optional) Enable the kubelet service before running kubeadm:

    sudo systemctl enable --now kubelet

  3. NTP setting

    • 클러스터의 서버들의 시간이 동기화 되지 않으면 해당 클러스터는 연결이 안 된다.

  4. DNS setting

    • 클러스터의 서버 OS 계열이 다를 경우 해당 DNS 참조 영역이 다를 경우 쿠버네티스 네트워크 연결이 되지 않는다.
    • DNS 설정이 /etc/resolve.conf일 경우
      • /usr/lib/systemd/system/kubelet.service 파일을 수정
      • /usr/lib/systemd/system/kubelet.service.d/10-kubeadm.conf 파일을 수정
      • 위 파일에 –resolv-conf=/etc/resolv.con 추가
    • systemd 데몬 재로드: sudo systemctl daemon-reload
    • kubelet 서비스 재시작: sudo systemctl restart kubelet
    • 상태 확인: sudo systemctl status kubelet
ExecStart=/usr/bin/kubelet $KUBELET_KUBECONFIG_ARGS $KUBELET_CONFIG_ARGS $KUBELET_KUBEADM_ARGS $KUBELET_EXTRA_ARGS --resolv-conf=/etc/resolv.conf

__Kubernetes Auth #

  1. X.509 Client Certs: X.509 인증서를 이용한 상호 TLS 인증
  2. HTTP Authentication: HTTP Authentication을 이용한 사용자 인증
  3. OpenID Connect: Google OAuth와 같은 인증 provider를 이용한 인증
  4. Webhook 인증: Webhook 인증서버를 통한 사용자 인증
  5. Proxy Auth: Proxy 서버를 통한 대리 인증
