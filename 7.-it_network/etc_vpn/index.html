etc_VPN

VPN #

Virtual Private Network

  • VPN이란 서울역에서 부산역까지 한번에 연결하는 터널을 의미한다. 그러므로 서울역에서 근무하는 근로자는 부산역으로 바로 이동하여 일을 할 수 있다.

  • 사설망: 특정 지역에서만 통신이 되는 네트워크를 말한다. 보통 보안이 철저한 군용 통신망과 기업 통신망을 가리켜 사설망이라고 부른다. 또는 폐쇄망이라고도 부른다.

  • 공중망: 전세계적으로 통신이 가능한 인터넷을 공중망이라고 부른다.

  • PN의 핵심은 바로 서울역과 부산역을 잇는 ‘터널링’ 기술이라고 볼 수 있다.

    • 예를 들어, 한국에서 한국식 전화번호 형식 192.168.0.X을 가지고 통화가 가능하지만, 미국에서는 미국식 전화번호 형식 10.10.10.X을 사용하여 한국과 미국은 일반적으로 통화가 불가능하다.

    • 하지만, 한국식 전화번호 형식 192.168.0.X를 교환국(터널링)에서 한국식 전화번호 ‘192.168.0.X’를 미국식 전화번호 ‘10.10.10.X’와 특수한 암호화 방식으로 매칭하여 미국과 한국이 통화할 수 있게 한다.

    • 즉, 한국과 미국을 통신하게 하는 교환국의 역할이 터널링과 같다고 할 수 있다.

터널링 # 


- 터널링의 이름 그대로 고속도로의 터널과 비슷한 원리이다.

- 터널을 시작점(서울)에서 ‘데이터’를 암호화하고, 종착점(미국)에서 ‘데이터’를 해독하여 데이터 통신을 한다.

PPTP #

(Point-to-point Tunneling Protocol)

  • 2계층 터널링 프로토콜

  • MS사에서 개발한 프로토콜로 페이로드(데이터)를 암호화하고 IP헤더로 캡슐화하여 전송한다.

  • 보안이 미흡하다.

L2TP #

(Layer 2 Tunneling Protocol)

  • 2계층 터널링 프로토콜

  • PPTP와 캡슐화 방식은 동일하지만 추가적으로 IPsec의 ESP를 도입해 보안을 제공한다.

L2F #

(Layer 2 Forwarding Protocol)

  • 2계층 터널링 프로토콜

  • 시스코에서 개발한 프로토콜로 UDP를 사용한다.

  • PPTP와 다르게 하나의 터널에 여러 연결을 허용한다.

  • 터널모드: 터널 모드는 IP 패킷 전체를 보호한다. 라우팅 정보를 위해 New IP Header가 생겼다. 그리고 IPsec header로 인하여 오리진 IP 패킷과 IP 데이터를 통채로 보호한다.

  • AH(Authentication Header) : IP 헤더의 일부, AH 헤더, IP 페이로드 부분에 대한 인증

  • ESP(Encapuslation Security): 원본 IP 패킷 전체에 대한 암호화와 ESP 트레일러에 대한 암호화를 하고, ESP 헤더, 원본 IP 패킷, ESP 트레일러에 대한 인증을 하는데 대표적인 암호화 알고리즘에는 DES, 3-DES, IDEA, SEED 등을 이용

GRE #

(Generic Routing Encapsulation)**

  • 3계층 터널링 프로토콜

SSL #

  • 4~7계층 터널링 프로토콜

  • 넷스케이프 사에서 만든 프로토콜로 웹서버와 웹브라우저 간의 안전한 통신을 목적으로 한다.

  • IPsec과 기능은 거의 동일하나 암호화 방식이나 하드웨어가 불필요한 점 등 차이가 있다.

SSTP #

(Secure Socket Tunneling Protocol)

  • 잘 알려진 SSL(Secure Socket Layer) 암호화 프로토콜을 활용해 통신하는 방식이다.

  • SSTP는 온라인 활동을 보호하기 위한 목적으로 마이크로소프트사에 의해 개발된 터널링 프로토콜

  • 윈도우 사용자들이 쉽게 설정할 수 있도록 Window 7, 8, 10에 기본적으로 제공되는 프로토콜.

  • SSTP의 경우 인터넷을 통해 오가는 데이터를 잘 알려진 SSL(Secure Socket Layer)로 암호화

  • SSTP는 강력한 암호화 절차를 활용하기 때문에 윈도우 운영체제에 기본적으로 탑재된 VPN 중 가장 강력한 보안을 제공.

  • SSTP는 대중적인 HTTPS 웹 커넥션을 통해 트래픽을 전송하기 때문에 PPTP나 L2TP와는 달리 트래픽이 쉽게 차단되지 않는다는 장점이 있습니다. 따라서 PPTP나 L2TP/IPSec의 대안으로 주목받고 있다.

OpenVPN #

  • OpenVPN은 오픈소스 소프트웨어를 활용해 개발된 터널링 프로토콜.

  • OpenVPN 역시 SSL(Secure Socket Layer)을 활용해 데이터를 암호화.

  • OpenVPN은 이외에도 다양한 암호화 절차를 제공.

  • OpenVPN은 오픈소스 생태계 특성에 따라 보안 전문가들이 주기적으로 잠재적인 보안 위협 요소들을 개선하고, 또 다양한 암호화 방식을 지원하기 때문에 높은 보안성을 지닌다는 장점이 있다.
