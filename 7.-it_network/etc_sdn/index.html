etc_SDN

SDN #

SDN(Software-Defined Networking)

처음 SDN 개념을 접했을 때는 너무 개념이 어려웠다.

하지만, 실무로 겪어보니 굉장히 쉬운 개념이었다.

한마디로 하드웨어 인프라(네트워크 장비) 와 통신하기 위해서 소프트웨어 기반 컨트롤러 또는 애플리케이션 프로그래밍 인터페이스(API)를 활용하는 네트워킹에 대한 접근 방법이었다.

SDN과 NFVO 개념이 혼동되긴 한다.

SDN #

네트워크를 프로그래밍 하는 것이 중점으로, 네트워크를 프로비저닝하고 관리하기 위한 외부적인 수단(중앙 애플리케이션 컨트롤러)이 핵심이다. (검색 자료를 보면 L2~L3 중심이다) 네트워크 추상화 (Network Abstraction)

NFVO #

네트워크 기능(L4~L7)을 가상화 하여, 이를 클라우처럼 배포하고 관리하는 것이 핵심으로, NFVO는 애플리케이션 기능이 중심이 아니라, 네트워크 기능을 가상화가 중심이다. 네트워크 가상화(Network Virtualization)

두 개념은 약간 차이가 있지만, 크게 보면 SDN이 포괄적인 것 같다. 그리고 두 개념이 완벽히 분리된 것이 아니라, 실제 업무에서는 두 개념이 하나의 기술로 구현되고 있다.

경험 #

기본적으로 이러한 개념을 차용하여 글로벌 네트워크 회사인 ARISTA는 스위치 네트워크 장비를 오픈소스 기반으로 개방하였다. 리눅스 모드로 들어가면 파이썬 코드를 이용할 수 있고, 이를 이용하여 기존의 네트워크 config가 아닌 코드로 작성할 수 있게 만들어 놓았다.

그리고 중앙 소프트웨어 기반의 컨트롤러(네트워크 중앙 통제 서버라고 불러도 될 듯)를 각 네트워크 장비와 연결하여, 네트워크를 분산하고 통제하며, 자동화하는 시스템을 구축할 수도 있다.

실제로 중앙 컨트롤러에서 네트워크 장비 Active-Standby 구조에서 active가 작동을 멈추거나, 인터페이스에 문제가 생겨 트래픽이 끊기면, 중앙 컨트롤러는 이를 즉시 감지하고, Active와 Standby를 바꾸거나 또는 문제가 생긴 인터페이스 연결을 끊고 다른 인터페이스를 활성화하여 트래픽 우회가 되도록 한다.

물론 이와 같은 일련의 자동화 과정은 파이썬이나 쉘스크립트로 코드로 작성해야 한다.

그리고 이 자동화 코드만 작성하면, 문제가 생겼을 때 새벽이나 주말에 출근하던 엔지니어들은 더 이상 개인의 시간을 반납하지 않아도 된다. 월요일에 출근하여 문제 사항을 확인하고 원인 분석만 하면 되니 말이다.

하드웨어에 종속되어 있는 네트워크 소프트웨어는 이제까지 발전이 없었다. 각 제조사마다 다른 config와 문법을 갖고 있었고, 호환도 좀처럼 되지 않았다.

기존 네트워크 단점 #

고객 입장에서는 정품 보다는 값이 싼 벌크 제품이나 비교적 가격이 싼 다른 제조사의 제품을 구매하여 기존 레거시 장비와 연동하지만, 이는 결국 호환 문제로 여러 이슈가 생긴다. 그리고 이러한 이슈로 인해서 가장 골머리를 앓는 것이 엔지니어들이다.

개인적으로 보면 제조사에서 일부러 호환이 안 되게 막아 놓은 것이라고 생각할 수 밖에 없다. 정품 구매 유도하도록 말이다.

SDN 장점 #

하지만, 이러한 문제로 인해 불만이 축척 되어 왔고, 결국에는 SDN과 같은 하드웨어 종속에서 벗어난 소프트웨어 기능이 중점이 되는 네트워크 개념이 쏟아지고 있다.

SDN은 네트워크를 기능 중점으로 분화시켜 전체 네트워크를 프로그래밍하고 제어할 수 있는 개념이라고 표현하고 싶다.

(참고로 OpenFlow 프로토콜와 같은 특정 인터페이스 기능을 제공해야 하며, 해당 제품에서 지원하지 않을 경우 해당 기능을 사용할 수 없다.)

  • 데이터 플레인: 실제 트래픽을 전송하는 기능 부분

  • 컨트롤 플레인: 트래픽을 어디로 전송할지 결정하는 기능 부분

  • 애플리케이션 부분: 전반적인 리소스 요청 또는 네트워크 관련 정보를 통신하는 기능 부분

결론 #

어떻게 보면 AWS와 GCP처럼 가상화 서버를 생성하고 이를 클라우드(중앙)에서 코드로 네트워크를 통제하고 관리하는 모델이라고 볼 수 있다.

새로운 기술이 매일 쏟아지지만, 그 원리는 기존에 있던 것에서 벗어나지 않는 느낌이다. 하나의 기술을 알면 다른 기술과 체인 되어 기술이 점차
