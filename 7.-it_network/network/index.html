Network

Chap01 Network #

  • 네트워크가 동작하려면, 통신 규약(프로토콜)이 필요하다.
  • 핸드폰 전화번호처럼 010-XXXX-XXXX와 같은 규칙을 통해 통신하는 것을 통신 규약이라고 하며, 네트워크는 이와 같은 규약을 7 계층으로 세부적으로 나누어 통신한다.
  • 통신 규약을 7계층으로 나누는 것을 OSI 7계층 참조 모델이라고 한다.
  • 컴퓨터 제조사나 네트워크 제조사, 어플리케이션은 이 OSI 7계층의 통신 규격에 따라 제품을 만들어 전세계와 인터넷 통신이 가능케 하고 있다.
  • TCP/IP 계층 모델은 OSI 7계층을 기능적으로 묶어서 표현했다고 말할 수 있다.

Chap03 게이트웨이 #

  • 게이트웨이(GateWay)
    • 네트워크 트래픽이 이동하는 문이라고 할 수 있다.
    • 네트워크 트래픽은 자신이 어디로 가야 할 지 방향을 잡을 때 게이트웨이(GateWay)를 보고 찾아간다.
    • 기본적으로 192.168.1.1과 같은 형태로 X.X.X.1로 게이트웨이 주소를 설정한다.

Chap04 브로드캐스트 #

  • 브로드캐스트
    • 브로드캐스트란 네트워크 장비와 연결된 모든 매체에 ARP라는 통신 프로토콜을 뿌리는 것을 말한다.
    • 기본적으로 192.168.1.255과 같은 형태로 X.X.X.255로 브로드캐스트 주소를 설정한다.
    • 브로드캐스트의 목적은 Mac테이블을 만드는 것이라고 할 수 있다.

Chap06 라우팅(Routing) #

  • 라우팅
    • 라우팅이란 일종의 네비게이션이라고 볼 수 있다.
    • 즉, 라우팅 테이블이란 정보를 참조하여 다른 네트워크와 통신하는 것을 말한다.
    • 라우팅은 크게 두 가지 종류로 나뉜다.
      • 동적 라우팅
        • 네비게이션이 알려주는 최적의 경로를 찾아 가는 것이라고 할 수 있다.
      • 정적 라우팅
        • 지도와 주소를 가지고 경로를 찾아가는 것이라 할 수 있다.

Chap11 URI #

(Uniform Resource Identifier)

  • URL - Uniform Resource Locator
    • 리소스가 있는 위치를 지정
  • URN - Uniform Resource Name
    • 리소스에 이름을 부여 • 위치는 변할 수 있지만, 이름은 변하지 않는다. • urn:isbn:8960777331 (어떤 책의 isbn URN) • URN 이름만으로 실제 리소스를 찾을 수 있는 방법이 보편화 되지 않음 • 앞으로 URI를 URL과 같은 의미로 이야기하겠음
  • userinfo
    • 거의 사용하지 않음. 이메일 주소 아이디
  • host
    • 도메인명 또는 IP 주소를 직접 사용가능
  • port
    • 일반적으로 생략, 생략시 http는 80, https는 443
  • path
    • 리소스 경로
  • query
    • key=value 형태
    • ?로 시작, &로 추가 가능 ?keyA=valueA&keyB=valueB
    • query parameter, query string 등으로 불림, 웹서버에 제공하는 파라미터, 문자 형태
  • fragment
    • html 내부 북마크 등에 사용
    • 서버에 전송하는 정보 아님
